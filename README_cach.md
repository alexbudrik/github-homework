# Задание 1: Кеширование
* Медленная загрузка данных
Если приложение каждый раз обращается к базе данных или внешнему API, это замедляет работу.
→ Кэш позволяет сохранять часто используемые данные (например, список товаров), чтобы не запрашивать их заново.

* Высокая нагрузка на сервер или базу данных
При большом количестве пользователей одни и те же запросы повторяются.
→ Кэш снижает количество обращений к базе, уменьшая нагрузку и предотвращая перегрузки.

* Задержки при работе с внешними сервисами
API сторонних сервисов могут отвечать медленно или нестабильно.
→ Кэширование ответов позволяет использовать уже полученные данные без повторных запросов.

* Повторные вычисления сложных операций
Некоторые вычисления (например, аналитика или генерация отчетов) требуют много ресурсов.
→ Результаты можно сохранить в кэше и использовать повторно.

* Ограничения (rate limits) внешних API
Многие сервисы ограничивают количество запросов в секунду/день.
→ Кэш помогает уменьшить количество вызовов и избежать блокировки.

* Улучшение пользовательского опыта
Быстрая загрузка страниц и данных делает приложение более отзывчивым.
→ Кэширование ускоряет отклик интерфейса.

* Работа при временной недоступности данных
Если основной источник данных временно недоступен (например, база или API упал),
→ кэшированные данные могут использоваться как «резерв».
